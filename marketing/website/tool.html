"The Tool"

"The Tool" has always been one of the parts of this effort. From the very get go, I wanted the application to be something that people could use for their scripture needs, independent of what I've done with it. YanCEyWare is a great application for Windows Mobile devices, and while I don't expect this particullar application to do the same things it does, creating a tool so people can import their own content into either new or existing databases is still a part of this project. If there is a place where help could be used, it's with developing this tool.

Tool Specification/Story

During the process of writing the Scripture application I also worked on some code that would import the scriptures from the LDS Church's website. It was intended to be a one-time shot and so I did not take a lot of thought into making it clean or in choosing a technology that would be very universal. Instead I used something that was easy to use and work with, and I ended up leaving it alone after I found the Mormon Documentation Project. I haven't done anything with it since. I started a Java based project for the import tool, but there isn't anything worth saving there either. So feel free to start from scratch. If you want the original code I worked on (it's ruby code using Active Record, hpricot, and builder) you are welcome to take a look at it, however, it's very ugly as I wasn't even looking to distribute it.

Here are the features for a first release:

  • Command-line interface (would make it easy for scripting by other people, and possibly a web based client at some point)
  • GUI Interface (Easy for users to use)
  • Convert THmL
  • Convert OSIS
  • Should be able to import into existing database, as well as create entirely new database

THmL and OSIS

There is a lot of scriptural information and translations out there that use THmL and OSIS, and we can leverage these sources as both test cases as well as final results for publishing to the scriptures.digitalbias.com website. The idea is that there are a lot of people out there that want other content, like the bible in the ASV translation or they want the LDS Hymns. We can provide these to them without requiring them to create their own. If they want to import them into their own databases....they can use the tool. If they just want to use these databases and switch between different sources, they can do it without using the tool. Totally up to them.

If they want to add their own content, like say lectures or other scriptural based commentaries they could mark them up in THmL and then import them into their databases. It would be nice to have the tool convert from HTML into THmL, but with the less structured format of HTML it would be quite the undertaking to do so and may not be worth the effort as there may be other software programs out there (like eSword) that may do the same thing already. If you want more content, GoBible has quite a few translations, most of which are THmL.
