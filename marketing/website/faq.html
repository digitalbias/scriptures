F.A.Q.

Your common questions with their answers.

Question: I have just downloaded the KJV bible [or LDS Scriptures] but it won't open the database. Could you help me get it working? Thanks.
or
The idea is good but it repeatedly failed during download.
or
It does not download the database all the way. It says it is complete around 30% then when you go to choose the database it says it is invalid.

Answer: In most cases these problems are related to the amount of time it takes to download the initial scripture database. While some effort has been made to decrease the size of the files, both the KJV and the LDS versions are still large files (2.5 Mb and 3.6 Mb) to download to a phone. With each successive release of the application an improvement has been made to the download process, still more can be done on this front. But where does that leave you now?

There are a couple solutions.

First: Instead of letting the screen go dark while we wait for the download to complete, you can occasionally touch the screen and wait for the database to complete the download process. While it does take time, rest assured that if you just let it go, it will download it.

Second: This solution is related to the first: Get on a wifi network or connect to a 3G network and then start the download. It will go faster and will have less chance of the screen going dark while you wait.

Third: Don't use the phone to download the scripture database. You can download the database you want from the downloads section on this website.

Some technical details: When the download screen in the Scripture application is first opened, it reads an xml file [this xml file] for the location of the database and the name it should display. When the user selects the database in the list, it then opens a connection to database location and downloads the database to the phone. It then takes the zip file and extracts the actual database file from the zip and places it on the sd card in the scriptures directory. So, while the progress bar may show that the download has completed, it will take some time for the database to be extracted to the sd card.

There are a couple reasons for this solution.

  1. The XML file can be updated without having to distribute another version of the application.
  2. The scripture databases themselves can be distributed in different locations on the web without putting too much burden on one single server.
  3. Trusted sources could be added to the XML file and their content could be included in the download listing.
  4. The compressed files reduce the size of the download and hopefully decrease the time it takes to download.

Question: I'd love to see the NIV added to the downloads if possible.

Answer: I would too. There are a great number of translations of the bible out there, some of which are still under copyright. One of these is the NIV (New International Version). When Scriptures was first concieved and created the goal was to provide an easy way for people to use whatever translation of scriptures they preferred. In the case of the NIV, Zondervan, the company that holds the copyright for the translation, charges and up front cost of $7,000.00 and an additional $7.50 for every electronic version of their bible that is distributed. Based on these costs, there will not be a NIV version provided any time soon.

Question: I'd love to see the [insert favorite translation or church manual here] included as a download.

Answer: There are a number of translations and manuals that I would like to see included in the downloads as well. The biggest need with the application, besides coding the application itself, is "The Tool". "The Tool" has always been one of the parts of this effort. From the very get go, I wanted the application to be something that people could use for their scripture needs, independent of what I've done with it. YanCEyWare is a great application for Windows Mobile devices, and while I don't expect this particullar application to do the same things it does, creating a tool so people can import their own content into either new or existing databases is still a part of this project. If there is a place where help could be used, it's with answering common questions and with developing this tool.
